# 旅游助手数据接入设计文档

## 项目概述

本项目为旅游计划生成助手，通过接入WeatherAPI.com天气数据API，实现从数据获取到智能响应的完整闭环。

## 一、数据源选择

### 1.1 数据源
- **名称**：WeatherAPI.com
- **类型**：REST API（实时天气数据）
- **特点**：
  - 免费额度充足（100万次/月）
  - 数据全面（温度、湿度、风速、空气质量、紫外线等）
  - 稳定可靠（99.9%可用性）
  - 支持中文语言

### 1.2 接入方式
- **方式**：API定时拉取 + 用户查询触发
- **频率**：用户查询时实时获取，数据持久化存储
- **存储**：本地JSON文件持久化存储

### 1.3 关键字段

| 字段 | 说明 | 单位 |
|------|------|------|
| temperature | 温度 | °C |
| humidity | 湿度 | % |
| wind_speed | 风速 | km/h |
| pressure | 气压 | mb |
| precipitation | 降水量 | mm |
| visibility | 能见度 | km |
| uv | 紫外线指数 | - |
| condition | 天气状况 | 文本 |

## 二、数据流设计

### 2.1 数据流程说明

**数据流转过程：**

1. **数据获取阶段**
   - 用户通过对话界面查询天气信息
   - 系统调用 WeatherAPI.com 获取实时天气数据
   - 对返回的数据进行解析和验证

2. **数据存储阶段**
   - 验证通过后，将数据持久化存储到本地JSON文件：
     - `weather_history.json`：存储各城市的天气历史记录
     - `alert_history.json`：存储异常告警历史

3. **异常检测阶段**
   - 系统自动对新获取的数据进行异常检测
   - 检测规则包括：高温预警(≥38°C)、暴雨预警(降水量>10mm)、空气质量差预警(AQI≥150)、强风预警(风速≥50km/h)等8条规则
   - 检测到异常时生成告警记录并保存

4. **联动规则执行阶段**
   - 根据天气数据和检测结果，触发相应的联动规则
   - 联动规则包括：高温推荐室内景点、雨天调整行程、空气质量差建议佩戴口罩等6条规则
   - 生成智能旅行建议

5. **数据展示阶段**
   - 在天气看板(Dashboard)展示实时数据和统计信息
   - 显示活跃告警列表
   - 展示智能旅行建议
   - 支持自动生成周报和日报

**数据流向总览：**  
用户查询 → 天气API → 数据解析验证 → JSON持久化存储 → 异常检测引擎 → 联动规则引擎 → 数据展示层

### 2.2 核心数据文件

| 文件 | 路径 | 说明 |
|------|------|------|
| weather_history.json | data/ | 天气历史数据 |
| alert_history.json | data/ | 告警历史数据 |

## 三、异常检测规则

### 3.1 规则设计

| 规则ID | 规则名称 | 触发条件 | 严重程度 | 响应动作 |
|--------|----------|----------|----------|----------|
| R1 | 高温预警 | 温度 ≥ 38°C | 警告 | 推送防暑建议 |
| R2 | 暴雨预警 | 降水量 > 10mm | 危险 | 建议取消户外行程 |
| R3 | 空气质量差 | AQI ≥ 150 | 警告 | 推荐佩戴口罩 |
| R4 | 强风预警 | 风速 ≥ 50km/h | 警告 | 远离临时建筑 |
| R5 | 极端低温 | 温度 ≤ -10°C | 危险 | 防寒保暖建议 |
| R6 | 连续高温 | 连续3天 ≥ 35°C | 警告 | 持续防暑提醒 |
| R7 | 紫外线强 | UV指数 ≥ 8 | 警告 | 防晒建议 |
| R8 | 低能见度 | 能见度 ≤ 2km | 警告 | 出行安全提醒 |

### 3.2 告警数据结构

```json
{
    "rule_id": "high_temp",
    "alert_type": "高温预警",
    "city": "广州",
    "severity": "warning",
    "message": "广州当前气温高达38°C...",
    "timestamp": "2024-01-15T14:30:00",
    "data": {
        "temperature": 38,
        "humidity": 75,
        "condition": "晴"
    }
}
```

## 四、联动规则

### 4.1 规则设计

| 规则ID | 触发条件 | 联动动作 | 建议内容 |
|--------|----------|----------|----------|
| L1 | 温度 ≥ 32°C | 推荐室内景点 | 博物馆、购物中心、温泉 |
| L2 | 有降水 | 行程调整 | 取消户外改为室内活动 |
| L3 | AQI ≥ 100 | 健康建议 | 佩戴口罩、减少户外 |
| L4 | 温度15-28°C、无雨 | 户外推荐 | 适合徒步、拍照 |
| L5 | 极端天气 | 推迟行程 | 取消或改期 |
| L6 | 温差 ≥ 5°C | 穿衣建议 | 洋葱式穿衣法 |

### 4.2 建议验证机制

每次联动规则触发后，系统记录：
- 触发的规则
- 采取的建议
- 用户反馈（如有）

## 八、部署说明

### 8.1 环境要求
- Python 3.8+
- Flask
- requests库

### 8.2 API Key配置
1. 访问 https://www.weatherapi.com/ 注册
2. 获取免费API Key
3. 编辑 `config.py` 文件，替换 `WEATHER_API_KEY`

### 8.3 启动步骤
```bash
# 1. 安装依赖
pip install requests flask

# 2. 生成测试数据（可选）
python generate_test_data.py

# 3. 启动应用
python app.py

# 4. 访问应用
http://127.0.0.1:5000/dashboard
```
